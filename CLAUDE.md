## Project Overview

Interactive prototype of Tabby's in-app checkout "station screen" — the screen shown inside a merchant's app while a user completes their BNPL purchase in the Tabby app. Built to demonstrate exact screen behavior to developers.

## Commands

```bash
npm run dev      # Start dev server at http://localhost:5173
npm run build    # TypeScript check + Vite build (tsc -b && vite build)
npm run preview  # Preview production build
```

No test runner or linter is configured.

## Architecture

### State Machine

The app has 3 states defined as `StationState` in `src/types.ts`: `"sending"` → `"sent"` → `"complete"`. State is managed in `App.tsx` and flows down to all components. URL param `?state=sending|sent|complete` locks to a specific state and hides controls (used for Figma capture).

### Component Hierarchy

```
App.tsx (state + viewport scaling via useViewportScale hook)
├── PhoneFrame (iPhone bezel + notification overlay, hideNotification prop)
│   ├── StatusBar / SafariBar / NavBar (browser chrome)
│   ├── StationScreen (main content, pushes left for account/success screens)
│   │   ├── SpinnerIcon (rotating progress arc)
│   │   ├── Tracker → TrackerStep[] (timeline with animated fill-down lines)
│   │   └── Footer (account switcher + consent)
│   ├── AccountScreen (phone number input, slides in from right)
│   ├── SuccessScreen (purchase complete, slides in from right)
│   ├── TroubleBottomSheet (slides up, backdrop + drag-to-dismiss)
│   └── NotificationBanner (iOS notification, spring animation)
└── ControlPanel (3 buttons to drive state transitions, hidden when ?state param set)
```

### Key Patterns

- **Tracker step configs** are generated by `getTrackerSteps(state)` in `src/constants.ts` — each state returns different step data (colors, text, line fills)
- **Animations** use Motion library (`motion/react`, NOT `framer-motion`) for notification slide, indicator color transitions, and content expand (AnimatePresence). Spinner rotation uses CSS keyframes.
- **Screen transitions**: Account and Success screens slide in from right with spring animation; station screen pushes left 30% simultaneously. Bottom sheets slide up with backdrop.
- **Notification management**: `hideNotification` prop on PhoneFrame + `notificationDismissed` state in App.tsx. Bottom sheet dismisses notification on open; "Send notification" button restores it.
- **Account number change**: When phone number is changed via AccountScreen, state resets to "sending" to re-send notification to the new number. Same number keeps current state.
- **Viewport scaling**: `useViewportScale` hook in App.tsx scales the phone frame to fit any screen size
- **Static assets**: Vite `publicDir` is set to `assets/` — reference files as `/filename.png` (not `/assets/filename.png`)

## Tech Stack

- **Vite** + **React 19** + **TypeScript**
- **Tailwind CSS v4** — config via `@theme` block in `src/index.css` (not `tailwind.config.js`). Uses `@tailwindcss/vite` plugin.
- **Motion** (import from `motion/react`, NOT `framer-motion`)
- **Inter** font via `@fontsource-variable/inter`
